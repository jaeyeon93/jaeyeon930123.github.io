2017-12-07 TIL

자료형

  1. 어떤 유형의 데이터를 사용하는 것이지 실행 환경(컴파일러/인터프리터)에게 알려주는것.
  2. 소스코드를 바로 실행할 수 있는것을 인터프리터 언어 스크립트 언어라고한다.
  3. 컴파일러는 기계어를 번역해주는 역할을 한다.
  4. 자료형마다 사용 가능한 연살자(Operator)가 있다.

숫자의 연산자

  • +, -, %, /, * 5개의 연산자가 있다.

문자열

문자열은 “Immutable”의 특징을 가진다(문자열만 Immutable)
객체는 “mutable”이다.

불리언과 논리연산자

  • 논리연산자에는 && || ! 이 있다.
  • 논리연산자는 return이 true, false로 나오고
  • 비트연산자는 return이 숫자로 나온다.

이진수와 비트 연산자

  • And, or, not, xor

true & true
true && true

Var a;
Var b = 10;
Var c = 5.5;
Var d = “hello”;
Var e = “3”;

const : 안변한다, 읽기전용변수, 값을 바꿀 수 없을때
let : 선언을 2번 해주면 안된다, 할당은 여러번해도 되는데 재선언을 하면 안된다. 에러가 된다.
var에 비해 let const가 훨씬 더 안전하다.
const pi = 3.14;
let a = 5;

++z : 증가를 먼저하고 리턴

// >, <, ==, ~=, >=, <=, ===이 있음
var a = 3;
var b = 5;
var c = “5”;
a > b
a < b
b == c // 값만 비교
b == c // 데이터타입까지 비교
```

문자열의 길이 구하기

  • var a = “hello”;
  • a.length

객체와 문자는 다 참조방식으로 저장을 한다. 새로만들면 참조가 갱신이 된다. =을 사용을 하면 참조를 새로 만드는것이다.

undefined, null, NaN

  • undefined : 값이 정의되지 않았다, 값이 아직 할당 안된것이다.
  • null : 값이 비어있다는 것을 명시적으로 알림, 값이 아직 무엇인지 모르겠다는 의미이다. undefined는 값이 없다. null은 값을 잘 모르겠다라는 의미.
  • NaN : 값이 아니다. == 계산 불가능
