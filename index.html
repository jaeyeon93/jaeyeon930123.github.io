그림으로 공부하는 IT인프라 구조 (1) - 인프라아키텍처, 서버

IT인프라는 2가지 형태로 구성이 된다.

집약형 : 대형컴퓨터, 메인프레임 등 성능이 좋은것이고 집약형의 최대 장점은 단순하다는 점이다.

장점
  1. 한 대의 대형 컴퓨터만 있으면 되므로 구성이 간단.
  2. 안정성이 높고 고성능이다.
    단점
  3. 비용이 비싸다.
  4. 확장성에 한계가 있다.

분산형 아키텍처

장점
  1. 낮은 비용이다.
  2. 확장성이 매우 좋다.
단점
  1. 복잡하다.
  2. 한대가 고장나면 영향을 최소화하기위해 설계가 필요

클라이언트 - 서버형 아키텍처

장점
  1. 클라이언트측에서 많은 처리를 실행할 수 있어서 소수의 서버로 다수의 클라이언트를 처리할 수 있다.
단점
  1. 확장성에 한계가 필요

3계층형 아키텍처

3계층형은 ‘프레젠테이션 계층’, ‘애플리케이션 계층’, ‘데이터 계층’ 총 3가지 계층으로 분할이 되어있다.
프레젠테이션 계층
  • 사용자의 입력을 받는다.
  • 웹브라우저 화면을 표시한다.
애플리케이션 계층
  • 사용자 요청(request)를 처리
데이터 계층
  • 애플리케이션의 요청에 따라 데이터를 입출력
장점
  • 서버 부하 집중 개선
    단점
  • 구조가 클라이언트 - 서버 구성보다 복잡.

물리서버

서버는 렉(rack)이라는 것에 장착된다.
서버와 PC는 물리적으로 기본 구성이 같다.
CPU

CPU는 Central Processing Unit의 약자다. 서버 중심에 위치해서 연산처리를 한다.

CPU는 명령을 받아서 연산을 실행하고 결과를 반환한다.
CPU가 명령을 자발적으로 처리하는 것은 아니다. ‘프로세스’나 사용자의 입력을 받아서 그것이 OS에 명령을 내리고, OS(운영체제)라는 소프트웨어가 CPU에 명령을 내린다.
메모리
메모리는 CPU옆에 위치하고, CPU에 전달하는 내용이나 데이터를 저장하거나 처리결과를 받는다.

메모리에 저장되는 정보는 영구성이 없지만 액세스가 매우 빠르게 이루어 지기때문에 사용을 한다.
CPU자체도 메모리를 가지고 있다. 이것을 레지스터나 L1, L2캐시라고 부른다.

L1, L2 캐시에 있는 데이터는 L3에도 존재한다. 다른 코어는 자신 이외의 캐시를 확인하지 않고 L3캐시만 확인하면 되는 구조다.
메모리를 이용하려면 메모리 컨트롤러를 경유해서 CPU밖으로 나가야한다. 이때 처리지연을 줄이기 위해서 자주 사용하는 명령/데이터를 코어 가까운 곳에 배치를 한다.
버스
버스(bus)는 서버 내부에 있는 컴포넌트들을 서로 연결시키는 회선을 가리킨다. 버스에서는 버스가 어느정도의 데이터 전송 능력을 가지고 있는가(대역)이 중요하다.

알고리즘(1) - 약수의합 구하기

어떤 수를 입력받아 그 수의 약수를 모두 더한 수 sumDivisor 함수를 완성해 보세요. 예를 들어 12가 입력된다면 12의 약수는 [1, 2, 3, 4, 6, 12]가 되고, 총 합은 28이 되므로 28을 반환해 주면 됩니다.
  1. sumDivisor라는 함수는 num을 인수로 받는다.
  2. 약수의 합을 반환해주는 answer라는 변수를 선언해준다.
  3. 약수의 값들을 넣어둘 빈 배열 list를 선언한다.
  4. 약수는 1부터 해당수까지 1씩증가를 했을때 나눠지는 수가 약수라고 한다.
  5. i가 1씩 증가할때마다 num을 i로 나눠서 0으로 나눠떨어지면 그것은 num의 약수이다. 그 약수를 push()메소드를 이용해서 list에 넣어둬야한다.
  6. 약수들이 list라는 변수에 담겨져 있으면 이제는 그 안의 합을 구해야된다.
  7. for (var j of list)라는 함수는 list에 있는 각각의 요소들을 j라는 변수에 한번담고 다음변수로 넘어간다는 의미이다.
  8. j가 한바퀴 돌때마다 값을 answer에 더하고 마지막으로 돌면 그 합은 약수의 합이 된다.—

2017-12-08 Unix Chapter1

초보자를 위한 UNIX Chapter1 : Unix란 무엇인가?

Unix는 컴퓨터의 운영체제이다. 운영체제는 사용자와 상호작용하는 역할을 할 뿐 아니라, 컴퓨터상의 자원 관리 그리고 다른 컴퓨터와의 상호작용을 조율하는 소프트웨어이다.

Unix의 명령어들을 동사라고 생각하면, 명령어들의 옵션은 형용사가 되고, 파일이나 디렉토리의 이름은 명사가 된다. 이 명령어들의 조합되어 만들어진 일련의 작업은 파일을 실행시키는 문장이된다.

다중사용시스템이란?

  • 다중사용자 시스템의 목표는 해당시스템을 사용하는 모든 사용자가 마치 자신만의 단독 시스템을 사용하는 듯한 느낌이 들게 하는것이다.
  • 각각의 사용자에게 계정(account)이 부여되며, 각각의 계정에는 홈 디렉토리가 부여된다.

셸이란?

  • 셸이란 명령행 기반 사용자 인터페이스를 셸이라고 한다. 셸은 사용자가 입력한 명령어들을 해석하는 인터프리터이다.

man 명령어

  • man은 Unix manual의 약자이다.
  • man -k옵션을 이용해서 키워드를 검색할 수도 있다.


mkdir 명령어에 대해서 man을 입력하면 다음과 같이 나온다.

man은 명령어에 대한 모든 설명이 나와있지만 이와 간결하게 whatis라는 명령어가 있다.

마치며

기초용어정리

  1. 계정(account) : Unix 시스템이 각각의 사용자에게 부여하는 이름.
  2. 인자(arguments) : 컴퓨터에서는 주로 어떤 프로그램이나 함수에 전달되는 옵션이나 파일명이다.
  3. 명령어(command) : Unix 시스템에 설치된 각각의 프로그램들을 전부 명령어라고한다.
  4. man 페이지(page) : 모든 표준 Unix 명령어에 사용법과 기능을 명시한 문서이다.
  5. 다중작업(multitasking) : 하나 이상의 프로그램이나 작업을 동시에 실행할 수 있도록 해 주는 컴퓨터를 멀티태스킹 컴퓨터라고 한다.
  6. 경로명(pathname) : Unix 시스템안에는 여러개의 디렉토리와 그 하위 디렉토리들이 존재하는데, 특정한 파일을 찾을때 디렉토리들을 어떤 순서로 찾아야 하는지를 명시하는 문자열이다.
  7. 셸(shell) : Unix 시스템을 사용하려면 명령어를 명령어 인터프리터상에 입력해야 한다. 이 인터프리터를 Unix에서는 셸, 혹은 명령어 셸(command shell)이라고 부른다.

Using Method - Array

Array

  1. arr.length : 길이 반환

  2. Array.isArray() : 인자 객체가 배열이면 true, 아니면 false

  3. arr.concat() : 인자로 주어진 배열이나 ㄱ밧들을 기존 배열에 합쳐서 새 배열을 반환.

  4. arr.filter() : 제공된 함수로 구현된 테스트를 통과하면 새로운 배열이 생성

  5. arr.every() : 배열의 모든 요소가 제공한 함수로 구현된 테스트를 통과하는지 테스트

  6. arr.find(), arr.findIndex() : 조건에 맞는 배열 안의 값을 하나 반환

  7. arr.forEach() : 배열 요소마다 한 번씩 제공한 함수를 실행.

  8. arr.includes() : 배열에 특정 요소가 포함되어 있는지 여부를 확인해서 true or false를 반환.

  9. arr.indexOf(), arr.lastIndexOf() : 배열에서 지정된 요소를 찾을 수 있는 첫 번째 인덱스를 반환하고, 없으면 -1을 리턴한다.

  10. arr.join() : 배열의 모든 요소를 연결해 하나의 문자열로 만든다.

  11. arr.map() : 배열 내의 모든 요소에 대하여 제공된 함수를 호출하고, 그 결과를 모아서, 새로운 배열을 반환

  12. arr.pop(), arr.push() : 배열에서 마지막 요소를 추가하거나 제거한다.

  13. arr.shift(), arr.unshift() : 배열에서 첫번째 요소를 제거하거나 추가한다.

  14. arr.reverse() : 배열을 반전시킵니다. 첫번째 요소는 마지막 요소가 되며 마지막 요소는 첫 번째 요소가 됩니다.

  15. arr.slice() : 배열의 begin부터 end까지(end는 불포함)에 대한 shallow copy를 새로운 객체로 반환.

  16. arr.splice() : 배열의 내용을 추가/제거하는 데 유용
    arr.splice(m,n,c) arr배열 m번째 인덱스에서 n개를 제거하고 c만큼을 넣는다.

  17. arr.sort() : 배열의 요소를 적절한 위치에 정렬하고 배열을 반환.

  18. arr.toString() : 지정된 배열 및 그 요소를 나타내는 문자열을 반환합니다.

  19. arr.reduce() : 피보나치수열을 풀때 유용

String, Number, Method Object

JS String and Number Method

String Object

  1. str.length => length는 메소드가 아니 property이기때문에 ()가 없다. Int와 같은 숫자형들은 length가 적용안된다.

  2. str.charAt(index) => 문자열에서 특정 문자를 반환

  3. str.concat(string2, string3 ..) => 하나 혹은 더 많은 문자열들을 결합하여 새로운 문자열을 반환하는 함수 Concat보다는 + 연산자를 더 많이 쓴다.

  4. str.includes(searchString[, position]) => 문자열 내에 찾고자 하는 문자열이 있는지 확인 // true, false로 반환

  5. str.(last)indexOf(searchValue[, fromIndex]) => 문자열 내에 찾고자 하는 문자열의 인덱스를 반환

  6. str.slice(beginIndex[, endIndex]) => 문자열의 일부를 추출하면서 새로운 문자열을 반환

  7. str.split([seperator[, limit]]) => 문자열을 하위 문자열로 분리하여 String객체를 문자열 배열로 나눕니다.

  8. str.substr(start[, length]) => 문자열에서 특정 위치에서 시작하여 특정 문자수만큼의 문자들을 반환

  9. str.substring(indexStart[, indexEnd]) => 한 인덱스와 다른 인덱스 사이 또는 문자열 끝을 통해 문자열의 하위 집합을 반환.

  10. str.toLowerCase(), str.toUpperCase() or variable.toLowerCase()

  11. str.trim => 양쪽 끝에서 공백을 제거

Number Object

  1. number.isInteger(value) => 전달된 값이 정수인지 여부를 가려냅니다. True false로 리턴

  2. Number.parseInt(string[, radix]) => 문자열 인수를 형변환하여 지정된 진법이나 기본 타입으로 리턴

  3. num.toFix([digits]) => 고정 소수점 표기법을 사용하여 숫자를 반환

Math.Object

  1. Math.abs(x) => 절대값을 반환

  2. Math.floor(x) => 주어진 숫자 이하의 가장큰 정수를 반환(내림)

  3. Math.pow(x,y) => x의 y제곱수를 반환.

  4. Math.round() => 어떤 수와 가장 가까운 정수를 반환(반올림)

  5. Math.sqrt() => 어떤수의 양의 제곱근을 반환
